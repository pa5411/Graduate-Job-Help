CV

This is a document where the skills of the applicant are outlined from previous experiences. The document should demonstrate how well the applicant fits with the culture of the company and their ability to perform well in the role. Most applications require a CV document to be provided. A CV may be read by a human or scanned for key words or phrases and analysed using software. Note that there are a wide range of opinions as to how best to write a CV. This guide outlines just one approach.

Layout

General layout

The CV should be written in reverse-chronological order, i.e. with the most recent experiences at the top. They must include the first name and surname of the applicant as well as contact details such as a phone number and an email address. A link to a website such as a GitHub portfolio may be added where necessary. Avoid putting a date of birth, home address and any details about race, sexuality or religion. If putting a CV on an agency board or public domain, consider using secondary contact information, such as a second email address and phone number to avoid spam.

The remainder of the document must have a section which includes dated relevant examples from work experience as well as a dated outline of the applicant's education history. Other sections may be added depending on the individual and the nature of the role. This may include a background summary as well as sections for technical skills, publications and references. Refer to the company website as well as the job description for information on what is expected, otherwise use common sense. Human Resource staff from companies advertising non-research roles for example, are unlikely to read through a list of publications.

Format

A CV can be written in any notes or documents software such as Notepad, MS Word or LaTeX. However, since assessors may view the document using different software and platforms, it may be desirable for the document to instead be submitted using the .pdf format. If this format is used, make sure that the text is not embedded so that a scanning tool can read the text easily. In other words, when the document is opened in this format, ensure that the text can be copied and pasted easily and does not appear to behave as an image. Always check the final document before submitting, to ensure that the formatting has not changed. If changes are needed, make them using the original software package that was used to first create the document. Generally the following rules should be followed:

  • Always place the name and contact details at the top of the CV.
  • Use a large font size for the name such as 16 and a regular font size such as 11 for the remainder of the text.
  • Use typical fonts such as Times New Roman, Liberation Serif or Computer Modern.
  • Use black colour only.
  • Only names, headings and titles should be in bold.
  • Avoid using tables for formatting as this may interfere with the scanning tool.
  • Avoid any graphics including passport photos. Use a portfolio hosted on a website instead if necessary.
  • Avoid paragraphs and use bullet points instead, except for the background summary.
  • Ideally use one page but no greater than two pages, with the exception of applications to academic positions.
  • Avoid very narrow page margins.
  • Avoid boarders.
  • If there are gaps in the CV, then only use the year part of the date otherwise put both the year and the month.
  • Ensure the dates are listed either all on the left side of the page or all on the right side of the page.
  • Save the CV document as "YourNameCV".

Language style

  • Use plain simple English. Avoid text speak and American English.
  • Avoid most abbreviations unless they are universally known, e.g. MS Word.
  • Avoid using pronouns such as "I", "you", "we", "he" or "she" etc.
  • It is useful to start each bullet point with an action verb, such as "presented" or "created".
  • Use adverbs such as "carefully" and "accurately" sparingly throughout the CV.
  • Use numbers to quantify the scope of an achievement, e.g. "presented to 30 people".
  • Keep bullet points to one line if possible and two lines at most.
  • Try to ensure that the end of each bullet point extends beyond the half-way point of the page.

Content

Generally only content which is relevant should be included in any section of the CV. Content is relevant if it highlights soft or technical skills which are comparable to the values of the company as well as the person specification of the job description. It is important to stress that if an example is not relevant, then ideally it should not be included in the CV unless there is remaining space to fill.

Sometimes a skill which is not explicitly mentioned may still be useful to include. For example, teamwork skills are always useful, regardless of whether it is mentioned in the job description. An important part of writing a CV is therefore understanding those skills which are useful and extracting them from previous experiences.

Note that a CV will be assessed based on how well the skills and experiences it describes match the criteria set by the company. The job description and values of the company give the best indication of the criteria set by the company. This is why emphasis has been placed on highlighting relevant skills.

It follows that sending the same CV out for every job application should therefore be avoided. This is because the same CV will never be able to include the relevant skills which match well to every single job role to which has been applied. Instead, a "tailored" CV is preferred, which means that the CV has been adjusted to highlight skills mentioned in both the job description as well as the values of the company. Multiple applications to the same company should also each have tailored CV's because one role may differ substantially from another role.

With this in mind, it may be useful to have several "generic" CV's written for different industries which are then used as a baseline from which further adjustments can be made. This avoids the need to have to re-write completely the CV for every role.

As described earlier, the CV is split up into several sections. The general content of each section of the CV will now be described.

Background summary

This is usually a short paragraph which includes a description of all degrees obtained or in progress as well as their key modules or research elements. The summary may highlight main industry related interests backed up with evidence and indicate career goals. In total, the summary should not exceed more than a few sentences. This section is usually not necessary and its inclusion is optional, however if included, it should always be the first section of the CV after the name and contact details.

Education history

This section should include details about all the university degrees that have been obtained or are in progress, up to and including PhD level. All grades should be indicated where applicable. PhD graduates should write either the type of degree that is written on the transcript or the subject. The title of the thesis should usually not be included. A-Level qualifications (or equivalent) should be included along with their grades. Undergraduate and postgraduate students may wish to mention the number of GCSE's (or equivalent) and any relevant grades such as English and Maths. However PhD graduates may omit them unless requested otherwise by the company. Place this section either before the work experience section or after it. Do not include achievements or awards in this section. They should go in a separate section instead.

Relevant work experience

This section should include experience gained from any degree as well as extra-curricular activities such as sports, part-time work, internships, apprenticeships and volunteering. It is the most important section of the CV and therefore should have the most time dedicated to writing it. Each example of an experience should have a list of bullet points which highlight skills that are relevant to the role. It is usual to have between 2 to 4 bullet points for each example. Avoid a situation where one example has a significant number of bullet points compared to the other examples. The intention is to instead highlight a range of skills obtained from a variety of examples. Some ideas for demonstrating common skills are indicated below.

Examples of communication skills

  • Presentations given to an audience during university or conferences.
  • Reports or leaflets created for an audience or individual.
  • Sketches, diagrams or schematics created for an audience or individual.
  • Arranging and attending team meetings.
  • Customer-facing work experience such as retail, phone operator, crowd marshal etc.

Examples of attention to detail skills

  • Spotting and reporting a bug in programming software or a book.
  • Working with complex machinery or equipment.
  • Dedicating a period of time to complete a complex task.

Examples of management skills

  • Leading a group of people in a team project, sports team or work environment.
  • Creating tools such as Gantt charts to work more efficiently.
  • Taking responsibility for a project or a problematic situation.
  • Being a carer for an individual.

Examples of learning skills or innovation

  • Completion of in-person or online courses.
  • Improving an existing idea or product with quantifiable results.
  • Developing a new idea or product.
  • Learning and applying new skills for a project or work experience.

For jobs with a company, the title used in the CV should be the same title as outlined in the work contract. If several positions have been achieved for the same company, for example Junior Consultant, Senior Consultant and then Manager, then each position should be listed separately.

Individuals with little work or volunteering experience will instead need to be creative in demonstrating their experiences from university. For example, in Figure 1, a PhD student has listed two projects separately, labelled project 1 and project 2. Be creative but clear with project names. Useful names include "Team Project on Subject Y" and "Lab Experiment on Subject Z". In both cases, the full name of the company or university as well as its location should be provided. Names which already include the location, such as "Imperial College London", may omit the location.

Check that the dates during which any university experiences were completed occur during the date outlined for the university in the education section of the CV.

Interests or hobbies

Any interests related to the role may be included in this section, but it is important to support them with evidence. For example, an interest in programming may be supported by participation in an online course. Other examples of evidence include hobby projects with portfolios, membership of societies or events attended such as career events or lecture talks.

Skills

The skills section should be viewed as a short summary for an assessor to quickly determine the key technical (or software) skills of an applicant. It is usually placed either immediately after the background summary or immediately after the work experience section. Depending on the nature of the role, this section can often be omitted. However, for IT or programming roles, this section is useful to highlight efficiently core skills.

Sometimes the proficiency of each skill is stated, e.g. MS Word (Expert) although such terminology can be vague and misleading. It is therefore always best practice to demonstrate how each technical skill was used in examples given in the work experience section. This is indicated in Figure 1.

Avoid putting any generic soft skills such as "team player" or "leader". Licenses and certifications may be placed in this section or in their own separate section. For research roles, experiment or field techniques may also be placed here or in their own separate section. Examples of useful skills to include are:

  • Programming languages such as Python or Java.
  • Major (relevant) libraries such as Tensorflow.
  • Software tools such as MS Word, MS Excel and Adobe Illustrator.
  • Experimental/ Simulation techniques such as Solidworks or FEA.
  • Licenses and certifications (*).

References

Before using an individual as a reference, ask them directly and confirm that they are happy to provide a reference upon request. Do not make an assumption that this will be the case. When adding a reference section to the CV, for each individual, add their title, first name, surname and contact details. Avoid using the personal home address of a reference unless permission has been attained. Many companies do not ask for references and therefore this section is often not needed. However, when necessary, two or three references is usually sufficient. The references section should be the last section of the CV.
